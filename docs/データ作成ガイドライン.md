# データ作成ガイドライン

本ドキュメントは GSC データパック準拠のデータ作成に関するガイドラインであり、生成AIへのインストラクションとしても機能します。

既存の GSC データパック（ロボ基準・拡張基準）の全データ傾向を踏まえ、一貫性のあるデータ作成を行うためのポリシーを示します。

## 1. 共通ポリシー（ロボ・拡張共通）

### 1.1 命名・識別子

#### パイロット名（キャラクター名）

- フルネームは全角で記述する。姓と名の間は `＝` で区切る（`・` は使わない）
  - 例: `アムロ＝レイ`, `緋村剣心`, `リナ＝インバース`
- 愛称（表示名）は全角10文字以内とする
  - 「あだ名」「苗字」「名前」のいずれかを使用する
- 同一キャラクターの時期・状態違いは `()` で区別する
  - 例: `アムロ＝レイ(0079)`, `弓さやか(後期)`

#### ユニット名

- 愛称は原則としてフルネームを用い、全角16文字以内とする
- これはインターミッションで画像が表示されない場面でも同名キャラを区別しやすくするため
- ミドルネームは省略または頭文字を使用する
  - 例: `アーチボルト＝Ｉ＝ウィムジィ`

#### 武器名

- 全角12文字以内を推奨（最大14文字）

#### 画像識別子

- **ロボ基準**: 作品略称を使用（例: `G0079_`, `MGZ_`, `GET_`, `EVA_`）
- **拡張基準**: 3文字を基本として他と被らないものにする（例: `STF_`, `RKEN_`, `R1&2_`）
- 識別子の後にアンダースコア `_` を付け、その後にキャラクター/ユニット名をローマ字で記述する
  - パイロット画像: `{識別子}_{名前ローマ字}.bmp`（例: `G0079_AmuroRay.bmp`）
  - ユニット画像: `{識別子}_{ユニット名ローマ字}U.bmp`（例: `STF_RyuU.bmp`）
  - バリエーションは `()` で示す（例: `G0079_AmuroRay(P).bmp`）

#### 読み仮名

- ひらがなで記述する
- パイロット・ユニットともに設定する

### 1.2 キャラクターの格（ティア）の概念

キャラクターには作品内での「格」があり、それに応じてデータ上の実効性能に差をつける。

#### 格の分類

| 格 | 説明 | 例 |
|----|------|-----|
| 主人公格 | 作品の主人公・メインパイロット | アムロ、兜甲児、流竜馬、緋村剣心 |
| 準主役格 | ライバル・相棒・重要敵幹部など主人公に次ぐ重要キャラクター | 神隼人、シャア、セイラ＝マス |
| サブキャラ格 | 継続登場するレギュラーで、作品内に明確な役割を持つキャラクター | 弓さやか、早乙女ミチル、ハヤト |
| 端役格 | 名前はあるが戦闘的役割が薄い。ちょい役・数話登場の中間幹部・名前付きザコ将校など | 退場する中間ボス、名前のある一般パイロット |
| 名前なし（ザコ） | 一般兵・量産型敵機・モブパイロット。名前を持たない | 一般ジオン兵、量産型ザク搭乗員 |

#### 格と実効性能の関係

- **主人公格は実効性能が最も高くなるようにする**
  - パイロットステータスの合計値が最も高い
  - 特殊能力の成長が早い（低レベルから高レベルに到達）
  - 性格は「強気」「超強気」が多い（気力が上がりやすく必殺技を使いやすい）
  - 搭乗ユニットも高性能（高HP・高攻撃力・高運動性の両立）

- **準主役格は特定分野で主人公に匹敵するが総合力で譲る**
  - 一部ステータスは主人公級だが、合計値はやや劣る
  - 特殊能力は持つが主人公より成長が遅い
  - ユニット性能は主人公機に準じるが何かしらの弱点がある

- **サブキャラ格は継続登場するが戦闘的決定打を持たない**
  - SP構成が補助・支援寄り（激励・応援・幸運・信頼等）
  - 戦闘で決定打となるSP（魂・覚醒）を持たない
  - ユニット性能は主役よりやや劣るが専用機または特化汎用機を持つ
  - 性格は「普通」が多い

- **端役格は存在感は薄く、データとしての個性も最低限**
  - ステータスが全体的に低く、特筆すべき特殊能力を持たない
  - SPは非常に少ない（集中・必中程度か、なし）
  - ユニットは量産型または低性能な準専用機
  - 登場期間が短いため、プレイヤーへの印象は薄い

- **名前なし（ザコ）は明確に低性能**
  - 全ステータスが低く、パイロット個人の個性を出す必要がない
  - SP・特殊能力はなし、または最低限（1〜2種）
  - ユニットも量産型低性能機
  - 集団での消耗戦向けに設計する

#### 実効性能を決める要素（重要度順）

1. ユニットの最強武器の攻撃力 × パイロットの格闘/射撃値
2. 気力上昇しやすさ（性格 × SP構成）
3. 回避/被弾のバランス（運動性 × 回避値 vs 装甲 × HP）
4. 特殊能力による底上げ（NT、底力、不屈等）

---

### 1.3 性格の設定基準

| 性格 | 気力上昇条件 | 典型的な使用対象 |
|------|-------------|----------------|
| 超強気 | 攻撃で+2, 被弾で+1, 回避で+1 | スーパー系主人公（兜甲児等） |
| 強気 | 攻撃で+1, 被弾で+1 | 主人公・戦闘好きキャラ |
| 普通 | 攻撃で+1 | 一般的なキャラクター。平和主義・内省的な主人公格にも使用（例: ロラン＝セアック） |
| 弱気 | 被弾で-1 | 非戦闘員・子供キャラ |
| 大人しい | 被弾で-1 | 内向的・冷静なキャラ |
| 慎重 | 攻撃で+1（敵撃破時のみ） | 戦術家・参謀タイプ |

性格は原作キャラクターの精神的傾向を優先して決定する。主人公格だからといって必ず「強気・超強気」とは限らず、気力上昇のしやすさが実効性能に影響することを念頭に置く。

### 1.4 前提オプション

GSC データパックは以下の 3 オプションを前提として設計されている。

- **乗り換え**: パイロットはカテゴリが一致するユニットに乗り換え可能
- **アイテム交換**: インターミッション中にパイロット・ユニット間でアイテムを交換可能
- **合体技パートナー行動数無消費**: 合体技使用時にパートナーの行動数を消費しない

以下のオプションには **対応していない**（これらを前提としたデータ設計は避ける）:

- 2回行動能力使用
- 合体技斜め隣接可
- 防御力成長

補足として、議論上「攻撃力低成長」を前提に比較する場合がある。ただしこれは ON/OFF でデータの動作自体を大幅に変える性質ではないため、上記3オプションとは扱いが異なる。

### 1.5 GSC 固有の追加・変更 SP

本体ヘルプに記載された標準 SP に加えて、GSC データパックでは以下の SP が追加または変更されている。
SP の書式は本体ヘルプ準拠で扱う。

#### 追加 SP

| SP名（略称） | 消費 | 効果 |
|-------------|------|------|
| 介抱（介） | 15 | 指定した味方ユニットの状態異常を回復する |
| 絆（絆） | 50 | 指定したユニットのHPを全快する |
| 気迫（迫） | 60 | 自分の気力を+20 |
| 献身（献） | 30 | 指定したユニットのパイロットのSPを10回復 |
| 堅牢（堅） | 30 | 1ターンの間、受けるダメージを1/4に抑える |
| 心眼（心） | 30 | 必中・ひらめきが同時に発動 |
| 忍耐（耐） | 20 | 自分の受けるダメージを一度だけ10に抑える |
| 報復（報） | 50 | 受けたダメージを1.5倍にして返す（相手のHPは最低10残る） |
| 勇気（勇） | 90 | 加速・必中・熱血・ひらめき・幸運・気合・努力が同時に発動 |

#### 変更 SP（標準と異なる消費・効果）

| SP名（略称） | 消費 | 変更点 |
|-------------|------|--------|
| 再動（再） | 90 | 指定した行動済みユニットを行動可能にする。消費が標準より増加 |
| 直撃（直） | 15 | 敵の回避系特殊能力・Ｓ防御・サポートガード・バリアを無効化。効果拡張かつ消費減少 |
| 突撃（突） | 20 | 次の戦闘でマップ攻撃以外の長射程武器が移動後も使用可能。消費減少 |
| 見極め（見） | 10 | マップ攻撃による攻撃が敵のみにダメージ。消費減少 |

### 1.6 SRCシステム差を踏まえた再現方針

SRC は「ボード型SLG」「交互行動」「移動と攻撃を同一手番で解決」「反撃が常時発生可能」という構造を持つため、原作システムの挙動をそのまま持ち込むと過剰性能になりやすい。

- MAP 兵器の安易な標準装備は避ける（複数攻撃と反撃不能化が強く出やすいため）
- 「即死」「完全無敵」などは制約付きシステム前提の表現であることが多く、SRC ではデフォルメして再現する
- 複数作品共演時のバランスを優先し、必要なら遠射程通常武装などで代替する

## 2. ロボ基準固有ポリシー

### 2.1 パイロットステータスの数値・範囲

パイロットのステータスは6つの基本値と性格で構成される。

```
格闘, 射撃, 命中, 回避, 技量, 反応, 性格
```

#### 数値範囲の目安

SP 以外の 6 ステータスは**目安**であり、作品の個性・特化型・ジャンルに応じた逸脱は許容される。SP は格に応じた水準を守ることを推奨する。

| パイロットの格 | 格闘 | 射撃 | 命中 | 回避 | 技量 | 反応 | SP |
|---------------|------|------|------|------|------|------|-----|
| 主人公格 | 150-170 | 125-165 | 135-155 | 130-165 | 170-190 | 155-170 | 50-80 |
| 準主役格 | 135-165 | 120-155 | 130-160 | 120-160 | 165-185 | 150-165 | 40-60 |
| サブキャラ格 | 120-150 | 110-140 | 120-150 | 120-150 | 150-170 | 140-160 | 40-55 |
| 端役格 | 95-120 | 95-120 | 110-130 | 105-125 | 140-160 | 130-150 | 30-45 |
| 名前なし（ザコ） | 80-110 | 75-110 | 90-115 | 85-110 | 120-145 | 120-140 | 25-40 |

- **SP標準値は50**（名前付きキャラ中の最頻値）
- 主人公格のスーパー系パイロットは SP 60〜80 の高めの値をとる
- 端役格の SP 標準は 35〜40、名前なし（ザコ）は SP 30 以下または SP なしも多い

#### 格闘特化型・射撃特化型

パイロットの専門性により、格闘または射撃の一方が大幅に低くなる「特化型」が存在する。特化型では片方のステータスが通常範囲を下回っても、もう一方で補完していれば整合性がある。

| 類型 | 格闘 | 射撃 | 代表例 |
|------|------|------|--------|
| 格闘特化型 | 通常範囲内 | 80-110（大幅低下） | 格闘・剣道主人公、武術系 |
| 射撃特化型 | 100-130（大幅低下） | 通常範囲上位 | 宇宙艦艇パイロット、スナイパー系 |
| バランス型 | 通常範囲内 | 通常範囲内 | 標準的な主人公・準主役 |

格闘値と射撃値の **合計値** が格の指標として機能するため、合計が総合的に格に応じていれば問題ない。

#### 強敵ボス・ラスボス格の上限

主要敵組織のトップ・ラスボス格は一部ステータスが主人公格上限を超えることがある（技量190超など）。この場合も実効性能は戦闘設計で調整する（アクセスしにくい武器制限、配置等）。

#### SP構成の格による傾向（ロボ基準）

| SP | 主人公格 | 準主役格 | サブキャラ格 | 端役格 | 名前なし |
|----|---------|---------|------------|--------|---------|
| 魂 | △※1 | - | - | - | - |
| 覚醒 | ○ | △ | - | - | - |
| 熱血 | ○ | ○ | △※2 | - | - |
| ひらめき | ○ | ○ | ○ | - | - |
| 集中 | ○ | ○ | ○ | △ | - |
| 必中 | ○ | ○ | ○ | △ | - |
| 激励/応援 | - | △ | ○ | - | - |
| 祝福/幸運 | - | - | ○ | - | - |
| 信頼/友情 | - | - | ○ | - | - |

○：典型的に持つ　△：持つ場合がある　-：通常持たない

※1 魂はパイロットの総合最大火力を底上げする最強攻撃SPだが、スーパー系ユニットなど搭乗機の基本武器攻撃力がすでに高い設計の場合は持たせないことも多い。ユニット最強武器の攻撃力水準と合わせて判断する。

※2 サブキャラ格でも前線で直接戦う戦闘役割を持つキャラクターには熱血を持たせることが標準的。補助・支援専門キャラには持たせない。

### 2.2 ユニット（ロボット）ステータスの数値・範囲

ユニットのステータスは以下で構成される。

```
HP, EN, 装甲, 運動性
```

#### スーパー系ユニット

| 指標 | 典型範囲 | 備考 |
|------|---------|------|
| HP | 4,500-8,000 | 主人公機は6,000以上 |
| EN | 100-150 | |
| 装甲 | 1,200-3,000 | 主人公機は1,500以上 |
| 運動性 | 40-70 | スーパー系は低め |
| サイズ | L-M | Lが多い |

#### リアル系ユニット

| 指標 | 典型範囲 | 備考 |
|------|---------|------|
| HP | 1,600-4,000 | 主人公機は3,000前後 |
| EN | 40-100 | |
| 装甲 | 500-1,200 | リアル系は薄め |
| 運動性 | 65-120 | リアル系は高め |
| サイズ | M-S | Mが基本 |

#### 武器攻撃力の目安

| 武器種別 | 攻撃力範囲 | 備考 |
|---------|-----------|------|
| 牽制武器（バルカン等） | 700-900 | 弾数多、命中補正高 |
| 通常射撃（ライフル等） | 1,100-1,400 | 主力 |
| 格闘武器 | 1,200-1,500 | |
| 必殺技 | 1,800-2,500 | EN消費・気力制限あり |
| 最終必殺技 | 2,500-3,300 | 厳しい制限付き |
| 合体技 | 3,000-4,700 | 複数パイロット必要 |

### 2.3 ユニットとパイロットの関係性

#### 基本方針

- **パイロットは「搭乗するユニットのカテゴリ」を持つ**
  - 例: `ＭＳ`, `マジンロボ`, `ゲッターロボ`, `戦艦`
- **ユニットは「搭乗可能パイロット数」を定義する**
  - 1人乗り（標準）、複数人乗り（サブパイロット付き）
- **乗り換えオプション前提**のため、同カテゴリ内で乗り換え可能に設計する

#### 主人公機の設計指針

- 主人公パイロットの高ステータスと組み合わせて最大効果を発揮するよう設計する
- 専用機は他キャラが乗っても一定の性能を発揮するが、本来のパイロットが最も活きる構成にする
  - 例: ガンダムは高運動性＋ビーム兵器 → アムロのNT能力・高回避・高射撃で真価発揮
  - 例: マジンガーZは高HP・高装甲＋格闘必殺技 → 甲児の超強気・底力・高格闘値で真価発揮

#### サブパイロット・サポートパイロットの設計

複数パイロットが関わるユニットには、**サブパイロット**（戦闘能力あり）と**サポートパイロット**（専用フォーマット）の2種類がある。

---

**サブパイロット**（ゲッターロボ・グレンラガン等）

ユニットに同乗し、SPを提供する。戦闘計算はメインパイロットが担う。

- 戦闘ステータスはメインより概ね 5〜15 程度低めに設定する
- SP量はメインと合算されるため**メインより低めが基本**（サブが多いほど総SP量が増えるためバランスに注意）
- SP構成はメインが攻撃・回避系（熱血・魂・ひらめき等）、サブは支援系（激励・応援・信頼・幸運等）を中心にする
- `pilot.txt` ではメインのブロックに続けてサブのブロックを記述する（特別なセクション区切りは不要）
- 推奨ユニットのコメントでサブ配置を示すと分かりやすい（例: `#推奨ユニット：グレンラガン(サブ)`）
- `robot.txt` 側でパイロット名を指定した武器使用条件を設定できる（例: `-(ヨーコ)`）

---

**サポートパイロット**（チャム＝ファウ・リリス＝ファウ等、フェアリー・精霊・非戦闘員キャラ）

戦闘には参加せず、SPのみを提供する専用フォーマット。

- カテゴリに `(サポート)` または `(XX専属サポート)` を付ける（例: `ＡＢ(サポート)`）
- 地形適応は `DDDD`（自立移動不可）
- 全戦闘ステータスは `1, 1, 1, 1, 1, 1`（戦闘不能）
- **SP量は 20〜30 が標準**。
- SP構成は支援系を中心（信頼・激励・幸運・補給等）に、作品内での役割に応じたSPを持たせる
- `サポートLv1` 特殊能力を付けることがある（メインパイロットのSP回復等に影響）
- 専属サポートの場合、カテゴリで搭乗ユニットを制限する（例: `ＡＳ(エレ＝ハンム専属サポート)`）

#### 地形適応コード

ユニットの地形適応は4文字で表す（空陸水宇）。

```
A: 得意  B: 普通  C: 苦手  D: 不可  -: 移動不可
```

- 空戦型: `AA-A` (VF-1 バルキリー等)
- 陸戦型: `-ABA` (ガンダム等)
- 万能型: `AAAA` (マジンカイザー等)
- 水中型: `-ADA` (ゲッター2等)

## 3. 拡張基準固有ポリシー

### 3.1 パイロットステータスの数値・範囲

拡張基準はロボ基準と同じ6ステータス構成だが、等身大キャラクターのため一部傾向が異なる。

```
格闘, 射撃, 命中, 回避, 技量, 反応, 性格
```

#### 数値範囲の目安

SP 以外の 6 ステータスは**目安**であり、作品の個性・特化型・ジャンルに応じた逸脱は許容される。SP は格に応じた水準を守ることを推奨する。

| パイロットの格 | 格闘 | 射撃 | 命中 | 回避 | 技量 | 反応 | SP |
|---------------|------|------|------|------|------|------|-----|
| 主人公格 | 140-170 | 110-155 | 140-160 | 135-165 | 155-190 | 150-170 | 55-60 |
| 準主役格 | 130-160 | 105-155 | 130-160 | 110-160 | 155-185 | 150-165 | 50-55 |
| サブキャラ格 | 110-150 | 90-140 | 120-150 | 110-150 | 140-170 | 140-160 | 40-50 |
| 端役格 | 90-120 | 80-120 | 110-130 | 95-125 | 130-150 | 125-150 | 20-40 |
| 名前なし（ザコ） | 70-100 | 70-100 | 85-110 | 80-110 | 110-135 | 110-130 | 20-30 |

- 拡張基準では射撃値よりも格闘値が重視される傾向がある（等身大格闘が多いため）
- SP標準値は主人公は60、ダブル主人公などの例外を除いて原則として主人公以外は55以下、55も準主役格のみで他は50ないしそれ以下

#### RPG・TRPG系作品の注意

RPGやTRPGのように「多数の戦闘員が均等に活躍する」作品では、キャラクター全員が準主役格〜主人公格相当の数値を持つことがある。この場合も作品内での相対的な強さ・役割でステータスに差をつけること。全員が同水準になる場合は、実効性能（SP構成・武器攻撃力）で格を表現する。

### 3.2 ユニットステータスの数値・範囲

拡張基準では「ユニット＝変身後の姿」「パイロット＝キャラクター本人」という構造が一般的。

```
HP, EN, 装甲, 運動性
```

#### ユニット系統

### 3.3 ユニットとパイロットの関係性

#### 基本方針

拡張基準では「パイロット＝キャラクター本人」「ユニット＝戦闘形態」として設計する。

- **変身ヒーロー型**: ユニット＝変身後の姿（仮面ライダー、ウルトラマン等）
  - パイロットは変身前の人物
  - ユニットは変身後の能力体
  - 専用性が高い（乗り換え不可が多い）
  - 原則として変身後を基準にデータ化し、変身ギミックの実装は任意とする

- **格闘家型**: ユニット＝キャラクターの戦闘スタイル（ストリートファイター等）
  - パイロットとユニットが一体
  - ユニット名＝パイロット名
  - 完全専用

- **RPGキャラ型**: ユニット＝クラス/ジョブ（FFシリーズ等）
  - パイロットはキャラクター本人
  - ユニットはジョブや装備状態
  - やや汎用性がある場合も

#### 空適応の特殊ルール（等身大基準）

等身大基準ではロボ基準と異なり、飛行不可能ユニットでも空適応 B〜C を付与する。

これは以下の理由による:
- 飛行ユニットが限定されている
- 飛び道具を持たないユニットが多い
- 飛行ユニットとの交戦機会における不均衡の緩和

ジャンプ力などを参考に空適応値を決定してよい。

#### 宇宙適応（等身大基準）

等身大ユニットの宇宙適応は「宇宙服を自動的に着るもの」として扱う。アイテムで補わせることはプレイアビリティを著しく下げるため推奨しない。判断基準は「原作での宇宙空間での活動実績の有無」。また「異空間」の表現として宇宙地形が使われる場合も考慮する（例: ウイングマンの宇宙適応Aはこのため）。

#### MAP兵器の扱い（等身大基準）

元の作品で集団攻撃能力があっても、SRC では MAP 兵器を安易に付与しない。MAP 配置や1ターンの戦闘構造上、MAP 兵器は元のゲームより有利になりやすい。遠射程の通常武器で代替できる場合はそちらを優先する。

#### 格闘ゲーム系キャラクター

ストリートファイター・KOF 等の格闘ゲームキャラは見かけによらず「遠近両用型」（飛び道具+近接格闘）。

- 気弾などの飛び道具には基本的に `実` 属性を付ける（気弾タイプは `射撃` パラメータ適用）
- 投げ技・ガード不能技だからといって `接` 属性が付くとは限らない
- 投げ技の基本属性は `突`
- 飛び道具の反射再現は `実` 属性を対象とする

#### 剣使い系キャラクター（等身大基準）

侍・剣士・戦士など近接武器を使うキャラクターが銃弾・魔法を切り払う場合の再現書式:

```
銃弾: 阻止LvX=弾き落とし Ｂ銃 切り払い
魔法: 阻止LvX=魔法切り払い 魔術 切り払い
```

#### 武器属性の追加注意（等身大基準）

- 射程2以上の武装には `射` 属性は付けない（これは等身大・ロボ共通）
- `術` 技能を使用した場合、射撃パラメーターが「魔力」として扱われる（魔法使い・超能力者型に適用）

## 4. 属性・武器設計の共通基準

### 4.1 主要属性の判断基準

| 属性 | 付与基準 | 注意点 |
|------|---------|--------|
| 実 | 投擲兵器、爆発する弾頭、誘導兵器 | 砲弾・バルカン・散弾（ショットガン等）・一般的な機関銃/ライフルには付かない。鞭系を対空武装化する場合は武/突より実を優先することがある |
| Ｂ | 粒子ビーム、レーザー、光線 | ビームサーベル等の近接武器には付けない。地形適応: 粒子ビーム `AA-A`、レーザー `AACA`。原理不明の光線は制作者裁量。「○○光線」「××ビーム」名の武装には付けておくとプレイヤーに防御法が伝わりやすい |
| 重 | 重力・ブラックホール等の重力攻撃。拳や武器から重力を放射するタイプの近接攻撃にも付く場合がある | 重力で剣を作って切り裂く近接攻撃や、補助的にしか重力を使わない近接攻撃には付かない |
| 突 | 格闘攻撃（パンチ・キック等） | 投げ技も基本は突 |
| 武 | 切り払い可能な近接武器攻撃 | |
| 格 | 射程2以上で格闘値を適用したい武器 | 射程1には付けない。投げる武器には格を付けるのが基本 |
| 接 | 攻撃準備段階で相手の動きを封じてから攻撃する技（例: 超電磁スピン）。射撃+近接複合技で「射撃部分が切り払いを封じる」場合も付く | 切り払い・近接無効バリア対策として強力なため基準は厳格。投げ技や「防御ごと破壊する技」には付かない |
| 先 | カウンター技、牽制用武装 | 強力属性のため多用しない。戦闘機の機銃などは可動域や役割次第で付かない場合もある |
| 連 | 連射・複数回攻撃 | レベルが高いほど命中しやすい。牽制武器は連L10（10回判定）、主力は連L5前後。全段入ることが前提のコンボ技には付けない。武装の役割次第では条件を満たしても外す |
| 貫 | ①準備技で装甲を劣化させてから攻撃（劣属性も同時付加が一般的）②装甲を貫通する準備動作後にメイン攻撃③原作で防御無視が強調されていた武装④内部へ直接攻撃する武装 | 「徹甲弾」「貫手」など一点集中ダメージは高CTで表現。「どんな装甲でも切り裂く」描写は大攻撃力で表現し貫属性は付けない |

### 4.2 命中属性

Ｈ/追/誘は総称して「命中属性」として扱う。距離補正オプションを使わない場合でも、ECM や状態異常の撹乱への関与があるため有効。

| 属性 | 付与対象 | 注意点 |
|------|---------|--------|
| Ｈ | ミサイル、ホーミング系、軌道を任意変化させられる射出武器（ロケットパンチ等） | 魚雷・ホーミングレーザー等も対象。自己推進でも軌道を任意変化できない武器（ロケットランチャー等）や自己推進しない砲弾/機銃、投擲武器には付かない |
| 追 | AI搭載の自律判断可能な武装、随伴サポート機体が突進する武装 | |
| 誘 | パイロットの脳波/テレパシーで制御する武装 | サ属性がある場合は誘属性不要（サ属性は誘属性の上位互換） |
| 有 | 有線射出武器、ワイヤー系。鞭系武装に実属性を付ける場合も有属性を付ける | 突属性・武属性がある場合は有属性不要。有属性が付く場合は命中属性（Ｈ/追/誘）は付かないことがほとんど |

### 4.3 ステータス異常発生属性

混乱・麻痺・睡眠等のステータス異常を発生させる属性。回復手段が限られるため以下の点に注意する。

- **味方使用ユニット**: CT率を低め、使用条件を厳しめに設定する
- **敵専用ユニット（低技量ザコが搭乗）**: CT率を高めに設定してよい

**害・憑属性の特別注意**: 効果が永続するため一度発動すると勝敗が決まりかねない（害属性でEN=0になるとボスでも行動不能になる）。効果ターン数を `L3` 程度に制限することを推奨する。

### 4.4 元素属性

元素属性はファンタジー系作品や特撮怪獣の弱点再現に使用する。現在有効な元素属性は以下の8種。

`火` `水` `風` `地` `雷` `冷` `光` `闇`

#### 付与基準

作中で属性弱点・耐性が**明示されている**場合、対応する武装がなくてもユニット側に付与する。

- 設定レベルで特性が記述されている場合
- 気象条件によって著しくパワーダウン/アップした場合
- データ設計上オミットした武装の特性によって撃破された場合

以下は弱点・耐性に成り得ない:

- 「大気圏突入で燃え尽きた」等、作中の常識的な出来事を無理矢理こじつけた場合
- 主役機の防御描写（装甲値・SPで再現し、耐性として付与しない）

また、火炎放射等の元素属性対象になりうる武装で、原作中で攻撃特性が有効に使われていなかった場合は、属性付与を必須とせず制作者裁量で省略してよい。

#### 光属性の特殊用途

直接的に「聖なる力」でなくても、「特定の敵に有効打を与える武装」の再現として転用できる（例: ガンパレード・マーチのNEP、マクロス7の歌など）。

#### 妖怪・モンスター（等身大）の共通ルール

等身大データにおける妖怪・モンスター系ユニットには `弱点=光` を付与するのが共通ルール。
